# 连续性审计

## 工作流程

1. 读取 `trackers/continuity.csv`、`trackers/foreshadowing.csv` 和相关故事圣经、大纲文件。
2. 检查用户指定内容是否违背既有正史设定。
3. 区分硬矛盾、软风险和可选优化。
4. 发现新正史时，更新或建议更新追踪表。

## 审计维度

- 人物：动机、知识范围、关系状态、说话方式、阵营忠诚。
- 主角塑造：坚韧、稳重、努力、聪慧、持之以恒是否通过行动体现。
- 修行：境界、资源、限制、伤势、恢复、代价。
- 功法与战斗：已学功法、实际战力、身体消耗、术法限制。
- 净尘炉：神器阶段、材料要求、能力范围、使用代价、暴露风险。
- 宗门老祖：身份是否保密、帮助是否合理、是否过度替主角解决问题。
- 货币：灵砂、灵石、贡献、丹药、任务奖励、宗门等级、市场价格。
- 妖灵：不得过早揭露；区分妖兽、邪祟、妖灵、妖灵残痕。
- 地理：距离、行程、权限、地图升级。
- 伏笔：埋设位置、延期兑现、重复提醒、被遗忘的承诺。
- 防撞车：场景或金手指是否类似高风险模板。

## 输出格式

先列问题：

- 零级：致命设定矛盾或明显撞车风险。
- 一级：严重连续性、动机或战力问题。
- 二级：可修复的清晰度或节奏问题。
- 三级：可选润色。

然后给出简短修复方案，以及应新增的追踪表条目。
